# Testing Standards

1. Add unit tests for domain logic and boundary behavior.
2. Add integration tests for cross-service paths and side effects.
3. Add regression tests for every production bug fix.
4. Keep tests deterministic where possible.
5. Diagnose the real cause of a failing test before changing code or tests.
6. Do not weaken contracts or mask failures just to get tests passing.
7. Keep test code to the same clarity and maintainability bar as production code.
